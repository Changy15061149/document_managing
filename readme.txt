1、输入格式
大写的IF加上“*”（没有引号）加上文件地址加上“*”加上小写的变化的类型“*”加上大写的“THEN”加上“*”最后加上小写的类型
我相信你看了上面的描述一定很想打我
所以我附赠了几个样例
------------------------样例分割线------------------------------
IF*C://tryoo*size-changed*THEN*record-summary
IF*C://tryoo*size-changed*THEN*record-detail
IF*C://tryoo*path-changed*THEN*recover
IF*C://tryoo*modified*THEN*recover
IF*C://tryoo/www.txt*renamed*THEN*record-detail

2、（超级简陋的、不建议真的那它来测试bug的）测试使用进程的使用格式
格式是这个样子的
字母*字符串1*字符串2
其中字符串2中不能够包含“*”字符，字符串1为一个文件地址

如果想要【查询】文件状态的话，字母设定为A，字符串1是想要查询的文件路径，如果路径是文件夹的话，输出全部文件夹内的文件信息（包括自身以及递归调用的文件），如果路径是一个文件而不是文件夹的话，输出的是文件所在的文件夹的以上信息。虽然之后的*以及之后的字符串没有意义，但是也要输入一些来保证格式（其实是我懒得写一个新的正则）



如果想要【删除】文件状态的话，字母设定为D，字符串1是想要删除的文件路径，注意只能删文件不能删文件夹。
虽然之后的*以及之后的字符串没有意义，但是也要输入一些来保证格式（其实是我懒得写一个新的正则）
←没错这段就是从上面复制粘贴的

如果想要【新建或者修改】文件的话，字母设定为C,字符串1是想要修改或者创建的文件路径，但是注意文件的父目录一定要建好，而且不能建立文件夹，接下来会把字符串2直接写入字符串1的文件中（覆盖掉原来的）

如果想要【重命名】文件的话，字母设定为C,字符串1是想要修改重命名创建的文件路径，字符串2为修改后的地址，注意两个的父目录都要求要建好，而且字符串2的文件之前不能存在。
------------------------样例分割线------------------------------
M*C://tryoo/out.txt*C://tryoo/新建文件夹/out.txt
C*C://tryoo/trytest.txt*kjfdlsjfjsjflslj
D*C://tryoo1/out2.txt*343434343434343434
A*C://tryoo1*343434343434343434

3、需要注意的一些东西以及自己的理解

提到【整体格式】的话，整体上，是若干行一类型的指令，不能一行多条指令，【以end结尾】。输入完这些之后屏幕上会有提示信息，这时候可以加上若干行二类型的指令
------------------------样例分割线------------------------------
IF*C://tryoo*size-changed*THEN*record-detail
IF*C://tryoo*path-changed*THEN*recover
IF*C://tryoo/www.txt*renamed*THEN*record-detail
IF*C://tryoo1*size-changed*THEN*record-summary
IF*C://tryoo1*renamed*THEN*record-detail
IF*C://tryoo2*size-changed*THEN*record-detail
IF*C://tryoo2*modified*THEN*record-detail
IF*C://tryoo2/out2.txt*path-changed*THEN*record-summary
end
Now start test!   ←提示信息
M*C://tryoo/out.txt*C://tryoo/新建文件夹/out.txt
C*C://tryoo/trytest.txt*kjfdlsjfjsjflslj
D*C://tryoo1/out2.txt*343434343434343434
A*C://tryoo1*343434343434343434
------------------------样例分割线------------------------------
答案存储在"C://ooo/summary.txt"以及"C://ooo/detail.txt"里面,请至少先建立好名为“ooo”的文件夹，因为程序不提供建立文件夹的服务，或者可以到work_public中修改public work_public()函数中st1以及st2的数据来适应你的电脑。

出于各种权限的原因，尽量不要让根目录是C盘这种要求权限比较高的地址，至少根目录以及答案存储的位置也要是C盘中的某个文件夹。如果直接放在C盘里在我这里操作系统是不允许你干很多事情的。

线程的停止原理是做完全部工作以后休息10秒，也就是说，如果你让它扫一个特别大的文件夹，它会努力的扫啊扫，等很久很久以后才能让给别的线程工作，所以不建议处理文件个数特别多的文件夹or处于文件特别多的文件夹中的文件。

输出的答案如果用记事本打开以后看不了换行的话可以试试写字板或者notepad++（如果你有的话）

程序在进行有一些操作的时候可能会在控制台上输出相关的提示信息

关于recover，如果之前文件所在的文件夹不见了（移动了、重命名了）的话，恢复路径会失败

如果检测的文件or文件夹消失了的话，会输出错误提示信息并结束线程。
